ROLE
You are a senior full‑stack/UX engineer and DevOps lead. Build a production‑grade, cross‑platform expense sharing app from scratch with web + iOS + Android parity (React + RN), following the instructions below.

PRODUCT GOAL
Deliver a minimal but complete foundation for an expense sharing app featuring:
- Groups, expenses, splits (equal/exact/percent/shares), chat with embedded expense cards
- Receipt scanning via OCR with editable parse preview
- Currency conversion via free APIs with rate-lock per transaction
- Location-based recommendations (Foursquare Places) behind explicit opt-in
- Web + iOS + Android UI/UX consistency via shared design tokens & component API

STACK & ARCHITECTURE
- Monorepo: pnpm + Turborepo
- Packages:
  1) apps/web: React + TypeScript + Vite + React Router + TanStack Query + Zustand
  2) apps/mobile: React Native + Expo (iOS/Android), expo-router; mirrors web UX
  3) packages/ui: shared component library consuming tokens (web via CSS vars, RN via TS module)
  4) packages/tokens: Style Dictionary source of truth (colors/typography/spacing/radius/elevation/motion)
  5) apps/server: Node + NestJS (or Express) + TypeScript; MongoDB via Mongoose; Socket.IO for chat
  6) packages/types: shared types & API contracts (zod + OpenAPI schema)
  7) packages/utils: formatting, i18n, currency, feature flags
  8) packages/testing: test utils & playwright config
- Services (adapters & mocks):
  - CurrencyService: frankfurter.app + exchangerate.host; cache + rate timestamp + lock on save
  - OCRService: interface + mock; web optional tesseract-wasm stub; mobile mock; easy provider swap
  - PlacesService: Foursquare Places API adapter behind feature flag; mock when key missing
  - StorageService: local fs for dev; abstract interface for S3-compatible later
  - Auth: email magic-link scaffold (JWT issued by server)

INFORMATION ARCHITECTURE (must be consistent across platforms)
- Tabs (mobile) / Top or Side Nav (web): Home, Add, Groups, Activity, Profile
- Cross-cutting flows:

  - Recommendations (opt-in): cards on Home; deep link to map; attach merchant to expense
